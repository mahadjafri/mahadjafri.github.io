Text-to-SQL Copilot for VS Code

A VS Code extension that translates plain-English prompts into schema-aware, validated SQL — adopted by engineers and analysts across the company as their default way of starting a new query.

01The problem

Even experienced engineers spent disproportionate time recalling table schemas, join keys, and naming conventions for a sprawling warehouse. New hires felt this more acutely; senior analysts still felt it. The cost was measured in minutes per query, multiplied by thousands of queries a day.

02The approach

I built a VS Code extension that pairs a natural-language prompt with retrieval over the warehouse catalog. The model sees the prompt, the relevant schema fragments, and recently-used patterns — and emits SQL that's pre-validated against the catalog before reaching the user.

The extension feels like Copilot, but for SQL specifically — and unlike a general code assistant, it actually knows what tables exist.

03What I built

  • VS Code extension with inline-completion UX, side-panel chat, and explain-this-query mode
  • Backend retrieval over table metadata, column descriptions, and historic query patterns
  • Pre-flight validator that catches invalid table/column references before the user sees the suggestion
  • Usage telemetry pipeline to measure acceptance rate, edit distance, and saved time

06What I learned

Adoption beats accuracy. The version that shipped wasn't the most accurate model I tested — it was the one with the lowest-friction UX. Engineers tolerate a wrong suggestion if dismissing it is one keystroke; they abandon a tool if it interrupts their flow.
